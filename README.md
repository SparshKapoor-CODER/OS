# OS - Open Source Governance Case Study and Toolkit

This repository contains an academic case study and a small toolkit for evaluating open-source governance practices around copyright, contribution policy, and fork health.

### Governance and Audit Scripts

| Script | Purpose |
|--------|---------|
| `init_oss_project.sh` | Initializes a sample OSS project scaffold for governance experiments. |
| `prepare_fork.sh` | Prepares a fork-ready project structure. |

| `compare_fork_health.sh` | Compares ecosystem and activity indicators across forks. |
| `governance_review.sh` | Runs governance-oriented checks and generates review outputs. |

## Case Study Summary

The PDF analyzes how LibreOffice emerged from OpenOffice.org after Oracle's acquisition of Sun, with emphasis on:

- governance structure versus legal openness
- ethics of centralized copyright control
- when forking is a justified community response
- stewardship design principles for new OSS projects

## Quick Start

1. Clone the repository:

```bash
git clone https://github.com/SparshKapoor-CODER/OS.git
cd OS
```

2. Read the case study PDF:

- `CaseStudy_24BAI10017_SparshKapoor.pdf`

3. Run governance scripts (Linux/macOS/WSL/Git Bash):

```bash
chmod +x *.sh
./check_dco.sh
./audit_copyright_risk.sh
./compare_fork_health.sh
```
